# AQAS (注意力量化套利系统) 产品需求规格说明书 v1.0

## 1. 产品愿景与北极星指标 (Vision & North Star)

- **产品愿景：** 打造一个在零和博弈的社交媒体环境中，以获取高质量人类注意力为目标的 24/7 自主化高频套利机器。它通过不断的环境试错与自我重写，实现对人类注意力的“逆向工程”。
    
- **目标“用户”：** 系统的实际运行中**不存在人类干预**。人类（团队）的角色是“策略投资者”和“系统监工”，只负责设定风险偏好和查看资产（沉淀的模因特征与流量收益）报表。
    
- **北极星指标 (North Star Metric)：** * **净胜优势 (Net Advantage)：** 生成策略的最终综合得分（$R_{total}$）持续击败同期同类目的大盘均值。
    
    - **自动化演进率 (Self-Evolution Rate)：** 系统在无人工干预下，自主发现并固化有效新特征（JSON Schema 更新）的频率。
        

---

## 2. 核心业务流：四阶段套利生命周期 (The Arbitrage Loop)

这是研发团队需要实现的核心业务态，AQAS 的每一次运行都必须经历以下完整生命周期：

### 阶段一：T-1 异动感知与猎物锁定 (Differential Scraping)

- **需求描述：** 系统必须具备实时监控目标垂直赛道的能力，但绝不全量抓取。
    
- **验收标准：**
    
    - 系统需定时在后台静默运行爬虫网络（Apify/定制节点）。
        
    - **硬性过滤：** 仅抓取发布 24 小时内，点赞率/评论率偏离该博主历史均值 $+3\sigma$（黑马爆款）或 $-2\sigma$（异常扑街）的内容。
        
    - **降噪处理：** 调用本地小语言模型（如 Qwen2.5-7B），将复杂的 HTML/图文结构，剥离清洗为包含“文本骨架、核心视觉元素、前 50 条高赞评论”的标准化 JSON。
        

### 阶段二：T0 策略计算与沙盒博弈 (Strategy & Sandbox)

- **需求描述：** 接收到异动数据后，系统的主控中枢必须自主生成应对策略，并进行内部优胜劣汰。
    
- **验收标准：**
    
    - **记忆检索：** Planner Agent 必须先向向量数据库查询历史高权重的“多巴胺触发器（Meme Features）”。
        
    - **变异生成：** 根据当前环境的收益方差，动态读取 $\epsilon$（变异率）参数，生成 10 个以上带有一定反常识逻辑的图文策略组合。
        
    - **沙盒斩杀：** Monitor Agent 在内部执行 $R_{proxy}$ 打分（评估钩子强度、信息密度等），强制淘汰 70% 的劣质策略，仅保留 Top 3 进入执行层。
        

### 阶段三：T0 物理渲染与推流 (De-AI & Rendering)

- **需求描述：** 将抽象的 JSON 策略，渲染成具备极致“网感”的像素级图文，并突破平台的 AI 痕迹风控。
    
- **验收标准：**
    
    - **文本去 AI 化：** Actor Agent 必须严格遵循黑名单词库，动态调整句式熵值，生成符合人类碎片化阅读习惯的正文与标题。
        
    - **视觉无头排版：** Visual Agent 必须通过 ComfyUI 生成纯净底图，并编写/调用 HTML+CSS 模板，通过 Puppeteer 浏览器无头截图，严禁使用大模型直接生成带字图片。
        
    - **自动化发布：** 通过模拟器或逆向 API 将成品推入真实社交媒体流量池。
        

### 阶段四：T+15m / T+24h 环境反馈与自我重写 (Feedback & Evolution)

- **需求描述：** 闭环的最关键一步。系统必须根据真实反馈，自主修改底层的数据库和参数。
    
- **验收标准：**
    
    - **破冰回收 (T+15m)：** 探针必须在发布 15 分钟后抓取点击率与秒级停留，计算 $R_{hook}$ 奖励。
        
    - **价值定调 (T+24h)：** 探针抓取赞藏比，计算 $R_{deep}$ 奖励，核算最终的 $R_{total}$。
        
    - **元认知固化：** 如果策略获得超额收益，Planner 必须触发“对比特征发现”逻辑，将新规律写入向量数据库，并通过执行脚本自主修改 `MEME_STATE.json` 和底层 Prompt 配置。

## 3. 非功能性与风控需求 (NFRs & Constraints)

这些是系统存活的底线，研发团队在技术选型时必须强制遵守：

- **平台反制对抗 (Anti-Detection)：** * 一切通过无头浏览器进行的动作（爬取与截图），必须注入指纹混淆脚本（如 Puppeteer Stealth），规避平台的反爬虫风控。
    
    - 内容的发布频率和时间点必须引入随机数（Poisson Distribution），模拟真实人类作息。
        
- **毒性惩罚绝对阈值 (Toxicity Block)：** * 在沙盒预审（$R_{proxy}$）阶段，必须接入轻量级的不良信息过滤模型。一旦涉政、色情或严重违背公序良俗的分数超过 0.1，策略直接执行“一票否决”级销毁。
    
- **上下文内存防洪 (Context Defense)：**
    
    - 系统运行时必须监控 Agent 的 Token 消耗水位线。达到 90% 时必须无条件触发 wU2 压缩机制，将对话状态总结并存储，彻底清理冗余 log。
