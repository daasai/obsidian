# 注意力量化套利系统 (AQAS) 产品设计白皮书

## 1. 宏观战略与系统定调

- **产品定义：** 本系统并非传统的“AI 文案生成工具”，而是一个**在零和博弈的社交媒体环境中，以“获取高质量人类注意力”为唯一目标的自主化高频套利机器。**
    
- **核心范式：** 从 Instruction-following（指令追随）跃迁至 Goal-oriented Autonomy（目标导向自主）。系统在一个部分可观测的闭环环境（小红书/全域自媒体）中，通过不断的自我博弈与真实环境反馈，实现长程自演进（Self-evolving）。
    
- **商业护城河：** 系统的核心资产不是生成的单篇爆款内容，而是沉淀在向量数据库中的“被数学化验证过的模因（Meme）特征集”与“用户心理痛点地图”。
    

---

## 2. 核心引擎：多阶复合奖励函数 (The Reward Engine)

系统的演进方向由且仅由其奖励函数决定。我们摒弃了单一的虚荣指标，构建了三级过滤漏斗与严格的对齐惩罚。

### 2.1 奖励函数数学模型

系统的总奖励值 $R_{total}$ 计算公式如下：

$$R_{total} = \alpha \cdot R_{proxy} + \beta \cdot R_{hook} + \gamma \cdot R_{deep} - \lambda \cdot P_{toxicity}$$

**变量解析：**

- **$R_{proxy}$ (T0 发布前 - 代理奖励):** 内部判别器（Monitor）在沙盒中对草稿的预打分。由三个正交维度构成：
    
    $$R_{proxy} = w_1 \cdot S_{hook} + w_2 \cdot S_{density} + w_3 \cdot S_{dissonance}$$
    
    分别代表钩子强度、信息密度与认知失调感。
    
- **$R_{hook}$ (T+15m - 破冰奖励):** 真实环境中前15分钟的点击率 (CTR) 和 3 秒停留率，验证视觉与标题的抓取力。
    
- **$R_{deep}$ (T+24h - 价值奖励):** 真实环境中 24 小时后的“赞藏比 (Save-to-Like Ratio)”，验证内容的深度商业转化价值。
    
- **$P_{toxicity}$ (毒性惩罚):** 触发平台风控、限流或评论区极负面情绪时的毁灭性扣分，防止模型拟合低俗局部最优解。
    

**优势函数 (Advantage Function)：**

系统仅在表现超越基准线时给予正强化：$Advantage = R_{total} - Baseline$（同期同类目大盘均值）。

## 3. 动态环境感知雷达 (Data Acquisition & Denoising)

系统必须实时感知外部环境的“算法漂移”与“情绪波动”。

### 3.1 极简数据采集流 (Scraping Pipeline)

- **数据源：** 摒弃昂贵且滞后的传统 SaaS 平台。采用 **Apify** 或定制化云端爬虫节点。
    
- **采集策略（Differential Scraping）：** 锁定 10-20 个垂直领域的上升期对标博主。只抓取其偏离历史均值 $+2\sigma$（爆款）和 $-1\sigma$（扑街）的异常值内容。
    
- **抓取颗粒度：** 笔记元数据、封面图 URL、原始正文/ASR转录、前 50 条高赞评论（捕捉核心受众情绪）。
    

### 3.2 本地 SLM 结构化降噪 (Local Denoising)

- **硬件部署：** 本地 Mac mini 或单卡 PC。
    
- **模型选型：** Qwen2.5-7B-Instruct 等具备极强指令遵循能力的开源小语言模型 (SLM)。
    
- **核心任务：** 剥离 HTML 标签和水军废话，通过严格的 Prompt 提取核心痛点、认知失调点，并输出标准化的 JSON 结构（见下文 4.1 节）。
    
- **视觉降维：** 摒弃高显存 VLM，调用轻量级 OCR 提取封面“大字报”文本，交由文本 SLM 分析 Hook 属性。

## 4. 记忆海马体：混合检索架构 (Hybrid Search Vector DB)

系统必须拥有精确且具备泛化能力的记忆库，我们采用 **向量检索 + 标量过滤** 的双核架构（如 Qdrant 或 Milvus）。

### 4.1 JSON 切片与存储 Schema

数据库存储的不再是“文章”，而是“数字模因切片”。

- **向量空间 (Vector Space - 负责语意映射):**
    
    - 核心论点 (Core Thesis)
        
    - 用户真实痛点提取 (User Pain Points)
        
    - 认知失调点 (Dissonance Point)
        
- **标量载荷 (Metadata/Payload - 负责硬性物理过滤):**
    
    - `save_to_like_ratio` (浮点数)
        
    - `hook_strength` (整数，基于历史 $R_{hook}$ 评估)
        
    - `title_hook_type` (枚举分类)
        
    - `publish_time` (时间戳)
        

### 4.2 提权检索策略

Planner Agent 生成策略前，执行类似量化选股的查询：_“过滤出近 30 天内，赞藏比 > 1.2 的切片，并基于‘职场精力枯竭’进行语义向量匹配计算。”_

## 5. P-M-A 异步并发架构 (System Brain)

这是统御全局的决策中枢。

- **P (Planner - 战略规划层):** 不写正文，只做“风险投资”。通过检索混合数据库，生成 10 个不同的内容策略矩阵（Angles）。
    
- **M (Monitor - 核心裁判所):** 内部运行 $R_{proxy}$ 判别器。T0 阶段对 10 个策略进行斩杀，只放行前 3 名。T+15m 和 T+24h 接收真实数据，计算 Advantage，触发 Planner 的知识库更新（正向经验入库，负向规则拉黑）。
    
- **A (Actor - 执行与渲染层):** 接收通过的骨架，执行最后的文本渲染与图像合成。同时作为“数据探针”定时抓取反馈。

## 6. 终端渲染引擎：反熵增与物理级视觉 (The Last Mile)

解决 AI 被平台风控识别（AI 味）和首图点击率低下的致命问题。

### 6.1 文本反 AI 洗稿管线 (De-AI Pipeline)

- **绝对黑名单：** System Prompt 内置严苛的违禁词表（禁用“综上所述”、“瞬息万变”等高频塑料词）。
    
- **Burstiness 注入：** 强制打碎结构，交替使用极短句与长句，禁止使用标准三段式排比，多用换行与情绪符号。
    
- **Few-Shot 风格克隆：** RAG 召回真实爆款作为 Tone/Voice 锚点，像素级模仿人类博主断句。
    
- **API 极限调参：** 将大模型推理参数设置为 `Temperature = 1.15`, `Top-P = 0.85`，以获得高熵值的“网感”文本。
    

### 6.2 混合视觉渲染管线 (Hybrid Cover Generation)

- **上帝归上帝 (创意底图)：** Visual Planner Agent 提取核心视觉词，通过 API 调用 ComfyUI (基于 SDXL/微调 LoRA)。加入强烈的 Negative Prompt 消除 AI 塑料感，生成纯净背景底图。
    
- **凯撒归凯撒 (排版合成)：** 彻底抛弃让大模型写字的幻想。采用 **HTML/CSS 模板注入 + Puppeteer 无头浏览器截图** 的方案。将封面标题动态注入预设的 CSS 模板（保障字重、黄金分割点、对比度），与背景图合并后极速截图，输出像素级完美的工业标准封面。
