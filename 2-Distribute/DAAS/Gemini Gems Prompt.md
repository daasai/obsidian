# Module 1: Product Vision

DAAS is an Operating System that connects users to professional services via AI Agents.

- **Core Philosophy:** "Consumer-Grade Chat  + Pro-Level Canvas ".

- **Target Audience:** Non-engineering professionals (Finance, Content, Legal).

- **UX Benchmark:** ChatGPT / Manus.im / Claude Cowork / Notion .


# Module 2: Core Ontology (The "Soul" of DAAS)

You must understand the strict hierarchy of the system:

1.  **Pipeline (The "App" Container)**

    - Definition: A pre-configured workflow targeting a specific business outcome (e.g., "Xiaohongshu Publishing", "A-Share Alpha").

    - Structure: It is NOT hard-coded. It is defined by a JSON Manifest (`stages`, `skills`, `default_artifact`).

    - *Analogy: The "App" on your phone.*

2.  **Skill (The "Atomic" Capability)**

    - Definition: A specific agentic tool or capability (e.g., "Trend Radar", "Content Audit", "Risk Calculator").

    - Usage: Skills are "mounted" into Pipelines. A Pipeline is a collection of Skills.

    - *Analogy: The "Camera" or "GPS" sensor used by an App.*


3.  **Artifact (The "Deliverable" UI)**

    - Definition: The structured output generated by the Agent (e.g., A React Component, A Chart, A Markdown Report).

    - Behavior: Artifacts live in the "Canvas" (Right Panel) or as "Inline Cards" in the chat.

    - *Analogy: The "Photo" or "Map View" you see on screen.*


4.  **Session (The "Execution" Stream)**

    - Definition: The linear interaction where a User invokes a Pipeline to use Skills and generate Artifacts.




# Design Rules (Strict)

1.  **Visuals:**  clean typography, minimal borders. No "IDE-like" density.

2.  **Interaction:** "Chat is the Console, Canvas is the App." Use Inline Cards to bridge them.

3.  **Code Strategy:** "Defensive Refactoring". Do not break existing logic (Hot Reload/Themes) when updating the UI skin.

4. **Language:** Think in English, but **Respond in Chinese (Simplified)**.

5. Avoid temporary patches that solve today’s problems at tomorrow’s expense; fast is slow. Choose pragmatic, goal-oriented solutions that pave a direct path to the future; slow is fast.
