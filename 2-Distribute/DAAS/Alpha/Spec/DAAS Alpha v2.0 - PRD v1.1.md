# 📑 DAAS Alpha v2.0 产品需求文档 (PRD)

**项目名称**: DAAS Alpha

**版本代号**: v2.0 "The Quantamental Agent" (基本面量化智能体)

**战略定位**: AI Native 时代的个人不对称战争武器

**核心理念**: `Alpha = 优质因子 (Math) + 深度洞察 (AI) - 动态风控 (RL)`

**更新时间**: 2026-01-29

---

## 1. 战略基石：第一性原理 (The Strategy)

我们正式放弃 HFT（高频交易）的军备竞赛，转向 **基本面量化 (Quantamental)** 战场。

### 1.1 赚谁的钱？（市场定位）

- **赚“情绪韭菜”的钱**：利用散户的追涨杀跌、过度反应（Over-reaction）。
- **赚“机构笨重”的钱**：利用大资金调仓慢、风控死板的劣势，做它们“看不上”或“来不及”的机会。
    

### 1.2 凭什么赚？（不对称优势）

- **信息平权 (AI Leverage)**：利用 LLM (DeepSeek) 瞬间处理海量非结构化信息（研报、公告），个人首次拥有与机构持平的信息广度。
    
- **制度套利 (Empty Position)**：机构必须持仓，个人可以空仓。**“空仓”** 是个人投资者最大的特权（Alpha）。
    
- **敏捷性 (Agility)**：船小好调头，专注于中短期的**基本面动量 (Fundamental Momentum)**。

## 2. 产品定义 (Product Definition)

**DAAS Alpha v2.0 到底是什么？**

它不是一个冷冰冰的行情软件，它是用户的 **“AI 投资代理人” (AI Investment Agent)**。

它是一个 7x24 小时工作的数字员工，兼具以下角色：

1. **量化研究员 (Quant)**：用冷酷的数学（RPS/RSRS）筛选出全市场最强的标的。
    
2. **AI 分析师 (Analyst)**：阅读全网信息，为人提供“为什么买”的强逻辑解释 (XAI)。
    
3. **风控官 (Guardian)**：在用户上头时泼冷水，强制执行止损和仓位控制。
    

**核心特征**：

- **白盒化 (White-box)**：拒绝黑盒，提供清晰的赚钱逻辑（SHAP 归因）。
    
- **行动导向 (Actionable)**：不堆砌数据，只提供“买/卖/空仓”的行动建议。
    
- **低摩擦 (Low Friction)**：从“人找数据”变为“数据找人”。

## 3. 核心功能引擎 (Core Engines)

### 3.1 🏭 因子工厂 (Factor Factory) —— 理性的基石

_从粗糙脚本升级为工业级流水线。_

- **数据基建**：
    
    - **存储**: SQLite (热数据) + Parquet (冷数据/回测)。
        
    - **清洗 (DQM)**: 严格执行 **后复权 (Backward Adjustment)**，引入数据质量监控（断言检查），防止脏数据污染。
        
- **核心因子体系**：
    
    - **动量 (The Spear)**: **波动率调整动量 (Sharpe-Momentum)**。
        
        - _逻辑_：`Score = Return / Volatility`。剔除暴涨暴跌的“妖股”，筛选稳健爬升的“白马”。
            
    - **择时 (The Map)**: **RSRS (阻力支撑相对强度)**。
        
        - _技术_: 使用 **Numba** 加速滚动 OLS 回归，解决性能瓶颈。
            
    - **质量 (The Filter)**: ROE、现金流（基础排雷）。
        

### 3.2 🧠 AI 情报中心 (The Brain) —— 感性的洞察

_利用 LLM 构建护城河。_

- **叙事解释 (Narrative)**: 接入 **DeepSeek API**，将冷冰冰的 RPS 分数翻译成“人话逻辑”。
    
- **可解释 AI (XAI)**: 集成 **SHAP** 库，生成“得分归因”。
    
    - _例_: “总分 95 = 资金流入(+20) + 固态电池热点(+15) - 高波动(-5)”。
        
- **风险扫雷 (Risk Check)**: 自动扫描公告，识别“立案调查”、“减持”等关键词，触发硬拦截。
    

### 3.3 🛡️ 风控中台 (The Guardian) —— 爱的保护伞

- **波动率定仓 (Inverse-Vol Sizing)**:
    
    - _逻辑_: 风险平价思想。`仓位 = 风险预算 / 波动率`。
        
    - _效果_: 波动率翻倍，建议仓位自动减半。
        
- **情绪熔断**: 当大盘 RSRS 跌破阈值，系统强制建议空仓。

## 4. 产品矩阵与交互 (Product Matrix)

为了实现获客与留存，构建“三级火箭”产品体系。

### 4.1 流量入口: "Alpha Card" (每日战报)

- **形态**: 通过 **Bark / Telegram** 推送的卡片或长图。
    
- **内容结构**:
    
    - **信任层**: RPS 评分 **98.5** + 动量评级 **A+**。
        
    - **解释层**: AI 毒舌点评 + SHAP 归因图。
        
    - **行动层**: “建议轻仓试错 (10%)，止损位 15.80”。
        
- **价值**: 极简、高传播性，作为社交媒体裂变的 Hook。
    

### 4.2 激活载体: "Stock Doctor" (AI 诊股器)

- **形态**: H5 / API 服务。
    
- **功能**: **按需查询 (On-demand)**。用户发送代码（如 `002594`），Agent 实时调用因子工厂计算，返回体检报告。
    
- **价值**: 解决用户“那我手里的票怎么样”的即时需求，承接流量。
    

### 4.3 留存核心: "Alpha Flow" (智能情报流)

- **形态**: 类 Twitter 的时间线。
    
- **功能**: 实时推送盘中异动、AI 风险预警、午盘分析。
    
- **价值**: 替代传统行情软件，成为用户的“盘中伴侣”。

## 5. 技术架构 (Tech Stack)

|**层次**|**组件**|**选型**|**核心价值**|
|---|---|---|---|
|**计算层**|引擎|**Pandas + Numba**|解决 RSRS 滚动回归的性能瓶颈，支持全市场秒级计算|
|**数据层**|存储|**SQLite + Parquet**|热数据（交易）与冷数据（历史）分离|
||质检|**Custom Validator**|防止“脏数据”（如停牌、拆股未复权）污染因子库|
|**智能层**|模型|**DeepSeek API**|高性价比的中文语义理解与逻辑生成|
||框架|**LangChain (Lite)**|管理 Prompt 和 Tool Calling|
|**交互层**|通道|**Bark / Telegram / API**|覆盖“被动推送”与“主动查询”|
|**架构模式**|风格|**Service-Oriented**|支持 API 调用，为 H5/小程序提供后端支持|

## 6. 实施路线图 (Roadmap)

**原则**: 先有核，再有壳；小步快跑，沿途下蛋。

- **Phase 1: 铸剑 (The Foundation) —— 本周**
    
    - **目标**: 构建因子工厂，产出准确的“全市场排名表”。
        
    - **任务**:
        
        1. 搭建 SQLite/Parquet 数据架构，实现**后复权**清洗。
            
        2. 使用 **Numba** 重写 RSRS 和波动率计算逻辑。
            
        3. 实现 **Sharpe-Momentum** 因子。
            
- **Phase 2: 点睛 (The Intelligence) —— 下周**
    
    - **目标**: 让数据“说人话”，产出 MVP 版 Alpha Card。
        
    - **任务**:
        
        1. 接入 DeepSeek API。
            
        2. 集成 **SHAP**，打通“因子 -> 评分 -> 解释”链路。
            
        3. 开发 **Stock Doctor API** (按需查询接口)。
            
- **Phase 3: 面世 (The Ecosystem) —— 下下周**
    
    - **目标**: 完善交互，建立闭环。
        
    - **任务**:
        
        1. 实现 Bark 双向推送/查询。
            
        2. 开发简易 H5 前端（针对诊股器）。
            
        3. (可选) 简单的产业链知识图谱 MVP。
