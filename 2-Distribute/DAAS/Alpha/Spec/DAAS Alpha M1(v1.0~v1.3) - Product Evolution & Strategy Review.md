## 1. 序言：为什么开发 DAAS Alpha？

DAAS (Data-Driven Alpha System) Alpha 的诞生并非为了制造另一个炒股软件，而是为了解决主观交易中的三大核心痛点：

1. **选择困难：** 面对 5000+ 只股票，人脑无法遍历所有机会。
    
2. **执行偏差：** “计划你的交易，交易你的计划”知易行难，情绪总会干扰决策。
    
3. **风控缺失：** 在市场下跌时，人性倾向于死扛而非止损。
    

我们试图构建一套**“外挂大脑”**，用代码固化交易逻辑，用数据对抗人性弱点。

## 2. 迭代复盘：从脚本到终端的进化之路

### 🌱 v1.0：原型验证 (The MVP)

- **形态：** 本地 Python 脚本 (`run.py`) + Excel 输出。
    
- **核心策略：** **纯 RPS (相对强度) 策略**。
    
    - _假设：_ 强者恒强。买入过去 20 天涨幅排名全市场前 5% 的股票。
        
- **痛点发现：**
    
    - 操作繁琐，甚至需要手动去行情软件复核。
        
    - 缺乏“择时”，大盘暴跌时依然发出买入信号（头铁）。
        

### 🎨 v1.1：可视化与交互 (The Interface)

- **形态：** 引入 **Streamlit** 框架，构建 Web 界面。
    
- **核心功能：** **Hunter (猎场)** 模块上线。
    
- **策略迭代：** 加入 **量比 (Volume Ratio)** 因子。
    
    - _思考：_ RPS 高可能是散户乱炒，只有“RPS 高 + 量比放大”，才说明有机构资金进场接力。
        
- **里程碑：** 从“看 Excel”变成了“刷 Feed 流”，大幅提升了选股效率。
    

### 🧪 v1.2：回测与真相 (The Truth)

- **形态：** 增加 **Backtest (回测)** 引擎。
    
- **核心事件：** 2023 年回测数据揭示了**“动量策略的死穴”**。
    
    - _数据：_ 在 2023 年震荡下跌行情中，最大回撤高达 19%，胜率跌至 37%。
        
    - _反思：_ **“Alpha (超额收益) 无法抵抗 Beta (市场风险)。”** 在泥沙俱下的环境中，勤奋地选股反而是在勤奋地亏钱。
        
- **结论：** 必须引入大盘择时系统。
    

### 🛰️ v1.3：专业化与闭环 (The Platform) — _当前版本_

- **形态：** 架构重构。**FastAPI (后端) + React (前端) + SQLite (持久化)**。
    
- **核心升级：**
    
    1. **Dashboard (驾驶舱)：** 集成 **BBI 多空过滤器**。
        
        - _逻辑：_ 指数 < BBI 均线时，强制空仓/防守。
            
    2. **Portfolio (模拟盘)：** 实现了 T+1 交易规则和自动记账，从“猜想”走向“实战”。
        
    3. **Lab (实验室)：** 可视化验证策略有效性，不仅看收益，更看回撤。
        
- **成果：** 即使在 2023 年的恶劣环境下，通过 BBI 过滤，成功将回撤控制在可接受范围，并大幅减少了无效交易。

## 3. 深度解析：v1.3 核心策略逻辑

v1.3 的策略体系可以概括为 **“趋势跟踪 + 截面动量”** 的双轮驱动模型。

### 3.1 进攻端：RPS + 量比 (Hunter)

我们寻找的是**“正在发生的趋势”**，而非预测未来。

- **RPS > 85：** 筛选全市场最强的 15% 股票。这利用了**“动量效应”**，即上涨的股票由于关注度提升，有惯性继续上涨。
    
- **量比 > 1.5：** 确认资金热度。剔除缩量空涨（游资盘）和放量滞涨（出货盘）的噪音。
    

### 3.2 防守端：BBI + 止损 (Risk Control)

这是 v1.3 的灵魂，用于解决“震荡市亏钱”的问题。

- **宏观择时 (BBI Filter)：**
    
    - _规则：_ 只有当上证指数**连续 3 天**站稳 BBI 线之上，才允许开新仓。
        
    - _本质：_ 宁可错过牛市初期的 5%，也要躲过熊市主跌浪的 20%。
        
- **微观风控 (Stop Loss)：**
    
    - _规则：_ 个股亏损达到 -8% 无条件止损。
        
    - _本质：_ 承认错误，截断亏损，让利润奔跑。

## 4. 第一性原理：我们到底在赚谁的钱？

从金融行为学角度，DAAS Alpha 的收益来源并非“预测”，而是**对人性的套利**。

1. **赚“羊群效应”的钱：**
    
    - A 股散户众多，一旦趋势形成，跟风盘会蜂拥而至。我们在 RPS 85 时介入，本质上是在**埋伏即将到来的散户抬轿大军**。
        
2. **赚“犹豫者”的钱：**
    
    - 机构建仓具有滞后性。量比因子帮助我们识别机构动作，利用**资金惯性**吃掉主升浪。
        
3. **赚“处置效应”的钱：**
    
    - 大多数人倾向于“拿不住好股，死扛烂股”。系统强制我们持有强势股，克服了**“恐高”**心理，从而获得风险溢价。

## 5. 局限性与边界 (知彼知己)

必须向团队明确，v1.3 **不是万能印钞机**。

- **适用环境：** 牛市初期、中期，主线清晰的高流动性市场。
    
- **失效环境：**
    
    - **快速轮动市：** 今天涨 AI 明天涨医药。策略会被反复打脸（Whipsaw），导致胜率下降。
        
    - **流动性枯竭：** 熊市末期，量比失效，容易买到流动性陷阱。
        
- **应对方案：** 当实验室回测显示胜率 < 40% 时，人工干预暂停策略，空仓观望。

## 6. 未来展望：v1.4 (AI Native)

虽然 v1.3 已经是一个成熟的**“价量量化系统”**，但它依然是“盲目”的——它只看数字，看不懂逻辑。

**v1.4 的方向是引入 LLM (DeepSeek/GPT-4)：**

- **逻辑因子：** 让 AI 阅读公告和新闻，判断 RPS 上涨背后是“真利好”还是“纯炒作”。
    
- **语义风控：** 监控情绪面，当全网都在吹票时，通过 AI 提示“拥挤度过高”预警。
    


## 7. 附录：核心术语表 (Glossary)

为了确保团队在开发与策略讨论中使用统一的语言，以下是对文档中核心概念的标准化定义：

### 📊 量化指标类 (Quantitative Indicators)

- **RPS (Relative Price Strength / 相对价格强度)**
    
    - **定义：** 衡量一只股票在过去一段时间（如 20 天或 60 天）内，相对于全市场其他股票的涨幅排名。
        
    - **取值范围：** 0 ~ 100。
        
    - **DAAS 含义：** `RPS = 90` 意味着该股票的表现优于市场中 90% 的股票。这是我们判断“强者恒强”的核心依据。
        

![stock relative strength chart的图片](https://encrypted-tbn2.gstatic.com/licensed-image?q=tbn:ANd9GcR_hNVt--ZpxZAMKHDqlxyM7vcC3MKKEmsN03hYaxecqJUi4sdgnjy8SXl7zSj_yKfj_QKByBXvirT_M5Nbn50RtFnGhTLJrEiGA6X5M0TNGRYNxfU)


- **BBI (Bull and Bear Index / 多空指标)**
    
    - **定义：** 一种将不同周期的移动平均线（MA3, MA6, MA12, MA24）加权平均计算出的综合指标。
        
    - **DAAS 含义：** 我们的“牛熊分界线”。股价在 BBI 之上视为多头（Bull），之下视为空头（Bear）。它是 v1.3 版最重要的风控开关。
        
- **Volume Ratio (量比)**
    
    - **定义：** `当前成交量 / 过去 5 个交易日的平均每分钟成交量`。
        
    - **DAAS 含义：** 用于侦测“机构进场”。
        
        - `> 1.0`：放量（关注）。
            
        - `> 1.5`：明显放量（机构抢筹）。
            
        - `< 1.0`：缩量（人气不足）。
            
- **Alpha (α) vs. Beta (β)**
    
    - **Beta (β)：** 市场平均收益（随大流）。如果大盘涨 10%，你的账户也涨 10%，这是 Beta。
        
    - **Alpha (α)：** 超额收益（凭本事）。如果大盘涨 10%，你的账户涨了 15%，那多出来的 5% 就是 Alpha。
        
    - **DAAS 目标：** 在吃满 Beta 的基础上，通过选股获取 Alpha。
        

### ⚙️ 交易与风控类 (Trading & Risk)

- **Max Drawdown (最大回撤)**
    
    - **定义：** 在选定周期内，账户净值从“最高点”下跌到“最低点”的幅度。
        
    - **DAAS 含义：** 衡量策略最坏情况下的风险。v1.3 的目标是将回撤控制在 15% 以内。
        
- **Whipsaw (锯齿效应 / 骗线)**
    
    - **定义：** 在震荡市中，价格刚发出买入信号随即下跌触发止损，导致“左右挨耳光”的现象。
        
    - **DAAS 含义：** 动量策略的天敌。我们引入“BBI 3日确认”机制就是为了减少 Whipsaw 带来的磨损。
        
- **Mark-to-Market (盯市)**
    
    - **定义：** 每日收盘后，按照当前市场价格重新计算持仓价值和账户总资产的过程。
        
    - **DAAS 含义：** 每日下午 17:30 自动运行的 `daily_runner.py` 的核心任务。
        
- **T+1 (Trading Rule)**
    
    - **定义：** A 股特有的交易制度。当天买入的股票（T日），必须等到下一个交易日（T+1日）才能卖出。
        
    - **DAAS 含义：** 模拟盘必须严格遵守此规则，否则回测数据无效。
        

### 🧠 理论与心理类 (Theory & Psychology)

- **Disposition Effect (处置效应)**
    
    - **定义：** 投资者的非理性行为——“过早卖出赚钱的股票（落袋为安），却死扛亏钱的股票（由于厌恶损失）”。
        
    - **DAAS 含义：** 我们的策略是反人性的：强制止损（卖出亏钱的），让利润奔跑（持有赚钱的）。
        
- **Reflexivity (反身性)**
    
    - **定义：** 索罗斯提出的理论。价格的上涨会改变投资者的预期，吸引更多买盘，从而推动价格进一步上涨（互为因果）。
        
    - **DAAS 含义：** 解释了为什么 RPS 策略有效——我们在利用趋势的自我强化机制获利。
