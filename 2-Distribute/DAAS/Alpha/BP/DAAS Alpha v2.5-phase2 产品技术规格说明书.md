我们将系统重构为三个核心流水线，分别对应信息的**获取、过滤与执行**。

|**层级**|**名称**|**AI 核心能力**|**商业价值 (Why)**|
|---|---|---|---|
|**L1**|**全知感知层** (Omniscient Perception)|**多模态解析 + 知识图谱**|**信息优势**：挖掘别人看不到的关联与情绪。|
|**L2**|**认知决策层** (Cognitive Decision)|**因果推断 + 体制识别**|**分析优势**：剔除伪逻辑，只在胜率高的天气出海。|

## 2. 详细模块规格 (Module Specifications)

### 2.1 [L1] 全知感知层：Alpha 的挖掘机

#### 模块 A：多模态情绪解析器 (Multimodal Sentiment Parser)

- **目标**：解决“财报文本滞后”问题，从非结构化数据中提取先行指标。
    
- **技术栈**：Gemini Pro 1.5 / DeepSeek-V3 (Text) + Whisper (Audio)
    
- **功能逻辑**：
    
    1. **输入**：上市公司路演录音、高管访谈视频、工厂卫星图（可选）。
        
    2. **处理**：
        
        - **Tone Analysis**：分析 CEO 语调的 `Jitter` (抖动) 和 `Shimmer` (微扰)，判断其对业绩指引的真实信心。
            
        - **Visual Logic**：识别“库存堆积”或“门口排队”的物理图像信号。
            
    3. **输出因子**：`Mgmt_Confidence_Score` (管理层信心指数), `Real_Traffic_Index` (真实客流指数)。
        

#### 模块 B：动态知识图谱引擎 (Dynamic Graph Engine)

- **目标**：解决“直接受益股涨幅过大”问题，挖掘滞后的“影子股”。
    
- **技术栈**：Neo4j + GraphRAG + GNN (图神经网络)
    
- **功能逻辑**：
    
    1. **动态构图**：基于新闻流实时更新节点关系（如：`SpaceX` -> `供应商` -> `A股影子股`）。
        
    2. **多跳推理 (Multi-hop Reasoning)**：
        
        - _Event_：铜价暴涨。
            
        - _Hop 1_：铜矿股涨（已兑现，太晚）。
            
        - _Hop 2_：空调企业成本上升 -> 毛利下降 -> **做空空调股**（隐性 Alpha）。
            
    3. **输出**：`Hidden_Exposure_List` (隐性受益/受损清单)。

### 2.2 [L2] 认知决策层：Alpha 的过滤器

_这是本版本的核心升级，用于**大幅提升夏普比率**。_

#### 模块 C：因果推断过滤器 (Causal Inference Filter)

- **目标**：剔除“伪相关性”，防止过拟合带来的实盘亏损。
    
- **技术栈**：Microsoft DoWhy / CausalNex
    
- **流程**：
    
    1. **System 1 提案**：“发现因子：当‘超级碗’收视率上升，美股科技股上涨。”
        
    2. **Causal Check**：构建因果图，引入潜伏变量（如宏观经济周期）。
        
    3. **判定**：DoWhy 计算 `ATE` (平均处理效应)。如果效应接近 0，判定为**“伪相关”**。
        
    4. **动作**：直接丢弃该策略，不予回测，节省算力并保护用户。
        

#### 模块 D：市场体制识别器 (Market Regime Detector)

- **目标**：解决“策略在不适合的行情下失效”的问题（动态风控）。
    
- **技术栈**：HMM (隐马尔可夫模型) / VAE (变分自编码器)
    
- **状态定义**：
    
    - `Regime 0`: 强趋势/低波 (All in)
        
    - `Regime 1`: 震荡/均值回归 (高抛低吸)
        
    - `Regime 2`: 恐慌/流动性枯竭 (空仓/现金为王)
        
- **落地策略**：
    
    - System 2 在执行回测前，先询问 Regime Detector。
        
    - 如果当前是 `Regime 2`，强制将所有生成的卡片建议改为“空仓观望”或“购买货币基金”。

### 2.3 [L3] 精准执行层：Alpha 的变现器

#### 模块 E：影子工厂 (The Shadow Factory)

- **目标**：7x24小时空转，为用户生产“预制菜”。
    
- **升级点**：

    - **Polars 加速**：确保全市场 5000 只股票的回测在 3 秒内完成。
