# 贝易转设计细化

时间: 2025年7月6日 10:16

系统设计待优化

1. 需要提前开户，以支持下单并开户的交互时效。
2. 各银行的来账回盘时效，以及异常情况下空中付主动查询的时效
3. 直联行schame获取失败，是否需要降级到直达行，路由（ai-router）需要识别商户配置，并反馈直联、直达还是兜底决策
4. solution：订单、支付单，sky：匹配单、流水模型，如何兼容用户切换直连/直达方式、切换银行、以及其他变种操作。
5. solution：订单、支付单，sky：匹配单、流水模型，如何兼容订单模式和合同模式下，异常来账的退款、挂账、入账。
6. 银行列表页下支付单时，提示用户确认付款人信息。
7. 用户信息不完整，需要UI交互增加补齐逻辑
8. 开户银行限制：并发、开户失败（查冻扣）
9. 收款账号开立：
    1. 商户入驻时，用商户23要素资质调银行做开户（子户）验证。
    2. 产品投产时，定期/常态按银行并发要求，持续调用银行开户（子户），控制账号池满足水位要求
    3. 业务下单时，从账号池中获取「空闲」账号，用于当前订单认款。
    4. 业务订单失效时，账号回收至账号池，并将原有的支付单、匹配单清理干净，避免未来被错误匹配。
10. 白名单报备，中信可支持不做白名单控制，空中付自己验证「非白名单」来账是否退款。
11. 支付结果通知，支付单成功需要回调商户通知。
12. 用从银行跳回贝易转后，需要有「是否已支付」的提示，或是有明显的刷新「查询支付结果」
