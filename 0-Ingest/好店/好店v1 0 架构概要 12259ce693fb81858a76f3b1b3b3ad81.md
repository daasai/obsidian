# 好店v1.0 架构概要

Owner: Xavier Ma
Tags: Onboarding, Process

# 1. 目标

 发展阶段：模式探索，重点突破人店运营，快速迭代业务模式、标准打法和激励牵引。

- 时间：2024.11~2025.3
- 规模：
    - 人店动员：1城，门店≤200家，活跃商户：≤1万户
    - 交易规模：日均交易额2000万元，交易笔数40万笔，预估峰值TPS 1000

# 2. 范围

1. **关键点**：
    - 性能：当前系统满足1000tps要求
    - 迭代：重点在经纪人端、消费者端和商户端，具备快速迭代交付能力
    - 安全：合规、资金安全和风险交易防范
2. **功能范围：**
    - **基础支付**：（合作）满足社区商户***基础*** 支付需求，品牌改造、产品复测和细节优化
    - **人店作业**：聚焦基础聚合支付展业，支撑门店/经纪人***营销、促活、服务和激励***的***基础***功能
    - **营销导流**：跑通**支付营销、业务导流、效果评价**的闭环
    - **数据运营**：支持平台**运营策略、数据洞察**的落地
    - **风控防控**：资金、交易和舆论风险防控
