# 好店产品路线图

Owner: Xavier Ma
Tags: Onboarding, Process

# 产品概要

好店，是运营可能性的生意。

## 产品路线图

### **（一）好店 v1.0**

面向社区商户提供新一代的聚合支付产品。在启动消费者触达前，先验证商户侧业务、营销、产品和服务模式。

- **目标客户**
    
    社区周边小微商户
    
- **价值主张**
    - **面向商户**
        1. **服务保障**。关键服务标准（可设计成服务承诺）：支付稳定、费率透明、及时结算、快速响应。 
        2. **场景适配**。基于金控产品，完成功能优化和增补。
        3. **就近服务**。问题快速响应，服务及时上门，社区营销赋能。
    - **面向门店/经纪人**
        1. **商机回报。**好店带来C端流量和商机，可直接转化为一体三翼商机。
        2. **利润回报。**好店模式通过商户交易流水产生持续毛利收入，其中40%分润给经纪人。
        3. **销售赋能，**极简产品+关键话术，帮助经纪人低成本掌握好店BD方法。
        4. **服务赋能，**将支付产品促活、服务拆解&简化为可执行的标准化任务，即：完成销售和服务工作的“动作分解”，降低经纪人作业门槛。
        5. **管理赋能，**支持商圈→城市→总部的层级管理支持。
- **业务目标**
    - 阶段：模式探索，重点突破人店运营，快速迭代业务模式、标准打法和激励牵引。
    - 时间：2024.11~2025.3
    - 规模：
        - 人店动员：1城，门店≤200家，活跃商户：≤1万户
        - 交易规模：日均交易额2000万元，交易笔数40万笔，预估峰值TPS 1000
- **关键点**：
    - 性能：当前系统满足1000tps要求
    - 迭代：重点在经纪人端、消费者端，以及商户端的完善，具备快速迭代交付能力
    - 安全：合规、资金安全和风险交易防范
- **产品定位**
    - **基础支付**：满足社区商户***基础*** 支付需求
    - **人店作业**：聚焦基础聚合支付展业，支撑门店/经纪人***营销、促活、服务和激励***的***基础***功能
    - **营销导流**：跑通**支付营销、B2A导流、效果评价**的闭环
    - **项目运作**：支持平台**运营策略、数据洞察**的落地
    - **风控防控**：资金、交易和舆论风险防控
- **迭代计划**
    - v0.5版本（10/31前交付）：
        - 聚合支付品牌改造
        - 结果页导流经纪人
        - 交易数据同步、沉淀和分析
        - 基础的运营、风控策略数据看板支持
    - v1.0版本（1/24春节前交付）：
        - 经纪人激励和分润支持
        - 经纪人培训、进阶和考核
        - 营销广告素材、导流与效果检测
        - 商户服务工单流程
        - （种子）商户需求反馈、改进和评价闭环
        - ~~好店任务体系（经纪人、商户和检核端）~~
    - v1.x迭代（2/5~3/31）
        - 终端系统：
            - 好店独立收银台小程序（微信&支付宝），强化稳定性、性能和交互体验
            - 社区业态SaaS：餐饮扫码点餐、便利超市、理发店等，适配社区商户特点
            - 商户端APP更新，强化提醒推送、费率活动和账单分析。
            - 聚合商户管理：
        - 银行运营：衔接银行合作，活动申报、进件、开卡、额度、出金等实现线上化
        - 商机运营：
            1. 素材闭环：内容经纪人自管理（复用已有机制：码上有客、主页等），根据门店、经纪人、LBS、用户行为和偏好，自动生成导流素材（房源、资讯等），以提高用户参与度和转化率
            2. 数据闭环：上游微信/支付宝，下游贝壳APP、小程序的埋点数据闭环；实时跟踪广告表现，自动调整流量分配，优化广告素材，确保每一次投放都达到最佳效果
            3. 效果策略闭环：实时监控广告表现，自动调整预算分配和优化广告素材，系统保障广告投放最佳效果。
- **技术储备**
    1. **聚合支付系统开发（难度：低，工作量：高）**
        - 聚合支付技术：掌握多种支付方式的集成与管理。
        - 微信&支付宝小程序技术：包括开发框架、审核规则、上下游交互和数据埋点等
        - 聚合SaaS系统：为不同社区业态（如餐饮、超市）开发适配的SaaS系统。
    2. **数据同步与分析（难度：低，工作量：高）**
        - 数据同步技术：实现跨平台的数据同步、存储和查询，需支持高频大数据量。
        - 数据分析：包括奥丁和chatBI，数据挖掘和分析工具，用于对人店、商户、交易数据的分析和展示。
    3. **经纪人管理（难度：低，工作量：高）**
        - 激励与分润系统：设计和实现经纪人激励和分润机制，包括分润规则配置、分润计算、贝壳币/资金发放。
        - 培训与考核系统：开发经纪人培训进阶和考核功能。
    4. **商户反馈与改进（难度：低，工作量：低）**
        - 商户反馈系统：实现商户诉求反馈、改进和评价的闭环管理。
    5. **营销与广告**：**（难度：高，工作量：高）**
        - 广告投放和效果监测：包括广告系统前后端开发、埋点数据存储和分析、A/B测试、广告效果评估以及安全与隐私保护等。
    6. **安全与风控（难度：高，工作量：低）**
        - 支付安全：确保支付过程的安全性和数据保护。
        - 风控策略：基本的风控策略和实现方法。
    7. **合作银行对接**：**（难度：高，工作量：低）**
        - 银行系统集成：实现银行合作业务的线上化，包括活动申报、开卡和数据同步等。
- **资源投入**
    - 人员：产研4~6人，必要时考虑引入外包。
    - 基础费用：新增5.3万元/月

### **（二）好店 v2.0**

1. **业务目标**
    - 阶段： 规模复制
    - 时间：2024.4~2025.10
    - 规模：
        - 人店动员：6城，门店≤2000家，活跃商户：≤10万户
        - 交易规模：日均交易额2亿元，交易笔数400万笔，预估峰值TPS 10,000
2. **产品定位**
    - **好店会员**：联合贝壳大会员体系，完成好店C端会员体系的创建和运营。
    - **商户产品：**满足社区商户**场景化需求的产品套系，**具备收款、带客、卖货、管店综合功能，具备高频更新、灵活拓展的“**活的系统”**
    - **人店作业**：聚焦门店/经纪人的聚合支付+社区营销展业：
        - **好店服务**：***营销、促活、服务和激励*** 的*“**全线上化**”*功能
        - **社区运营：**包括会员、线下集市、线上电商等
        - **AI应用：**AI赋能经纪人作业效率，包括不限于营销助手、进件简化、客服分身等。
    - **营销导流**：在B2A基础上，跑通B2B、B2C、（P2B）营销闭环
    - **项目运作**：全面支持业务**策略、数据洞察**的基础上，借助AI和大数据提升运作效果和效率
        - **消费者运营**：消费者分类、分层和触达
        - **AIGC营销内容**：AIGC生产商户内容，并在主要流量平台投放，包括小红书、抖音、美团等
        - **商户交易洞察**：销售线索、活跃检测、经营助手等
        - **设备巡检**：设备停用、设备故障和设备异常（遗失、转移）等的检测
        - **任务分发**：基于数据洞察，自动生成经纪人作业任务，系统完成推送、检核、业绩确认和分润激励。
        - **机构活动路由**：银行及其他机构、平台的活动分发和运营
    - **风控防控**：资金、交易和舆论风险防控，从手工，到自动，再到AI辅助。
3. **资源投入**
    - 人员：产研8~10人，必要时考虑引入外包。
    - 基础费用：471万/年
